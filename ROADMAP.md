# 🍒 Cherry Law 开发路线图

> 专业的法律业务场景 AI 助手开发计划

---

## 📋 概述

Cherry Law 是基于 [Cherry Studio](https://github.com/CherryHQ/cherry-studio) 二次开发的专业法律业务场景 AI 助手。本路线图规划了项目的长期发展方向和短期迭代目标。

---

## 🎯 项目愿景

成为法律从业者最信赖的 AI 助手，通过智能化工具提升法律工作效率，降低专业门槛，让法律服务更加普惠。

---

### v1.1.0 - 架构与数据底座重构版

**状态**: 🚧 开发中  
**目标日期**: 2026 Q3

#### 核心架构演进 (P0)

- [x] **统一数据层迁移 (Breaking Change)**
  - [x] 配置与大模型提供商 (`SettingService`, `LlmProviderService`) 迁移
  - [x] 文件元数据 (`FileManager`, `db.files`) 脱离 Dexie 迁移至 IPC
  - [x] 会话与消息记录 (`DexieMessageDataSource`, `db.topics`, `db.message_blocks`) 核心迁移
  - [x] 知识库与词条 (`Knowledge`, `QuickPhrase`) 迁移
  - [x] 全面废弃渲染进程 Dexie (IndexedDB) 和 Redux-Persist
  - [x] 全面转向主进程 **LibSQL + Drizzle ORM** 架构
  - [x] 实现主进程到渲染进程的标准化数据同步流 (StoreSyncService)
- [x] **底层大模型引擎重构 (aiCore)**
  - [x] 基于 Vercel AI SDK v5 构建统一模型抽象层
  - [x] 清理旧版冗余 Provider，实现按需加载与动态导入
- [ ] **核心 Provider 迁移 (Phase 1: OpenAI)** ⏳ **部分完成**
  - [x] 新架构 Provider 定义（`packages/aiCore` 中基于 `@ai-sdk/openai`）
  - [ ] 迁移 `models()` 与 `getEmbeddingDimensions()` 至新架构（当前仍代理到旧版）
  - [ ] 验证流式响应、工具调用与 Token 计费的一致性
  - [ ] 完全废弃旧版 `OpenAIBaseClient`
- [ ] **OpenCode Agent 架构重构**
  - [ ] 基于 `@opencode-ai/sdk` 统一 Agent 生命周期管理
  - [ ] 支持多 Agent 协作模式与能力动态扩展
- [ ] **业务场景引擎** - 领域特定工作流编排
  - [ ] 可视化场景构建器
  - [ ] 场景模板市场
  - [ ] 条件分支与循环支持
- [ ] **智能内容选择增强** - 精准上下文理解
- [ ] **深度研究模式** - 多源信息自动整合

- [ ] **法律 Agent 深度定制**

  - [ ] Agent 长期记忆与办案风格设定
- [x] **知识库增强**
  - [x] 法律文档专用加载器与分块器优化
  - [x] 文档分块策略与搜索结果展示优化
  - [x] 知识引用上下文高亮与折叠展示
  - [ ] 法律法规知识图谱
  - [ ] 案例库智能检索 (Hybrid Search)

- [ ] **法规比对 Skill** - 新旧法律法规条文差异对比分析
- [ ] **类案检索 Skill** - 相似案例智能匹配与裁判观点提取

### v2.0.0 - 智能生态与插件化架构版

**状态**: 📋 规划中  
**目标日期**: 2027 Q1-Q2

#### 核心功能

- [ ] **深度研究 (Deep Research)**
  - [ ] 类似 OpenAI Operator / DeepSeek R1 的多步推理架构
  - [ ] 自动化信息检索与多源数据整合
  - [ ] 任务目标拆解与自我修正循环
- [ ] **全局记忆系统 (Memory System)**
  - [ ] 基于 RAG (检索增强生成) 的长期记忆
  - [ ] **混合搜索 (Hybrid Search)**：结合 LibSQL 向量搜索与 FTS5 全文搜索
  - [ ] **知识图谱 (GraphRAG)**：在 SQLite 中构建实体-关系三元组存储
  - [ ] **主动记忆 (Agentic Memory)**：由 Agent 自主决定关键事实的存储与更新
  - [ ] 用户偏好与知识自动提取
- [ ] **插件化系统 (Plugin System)**
  - [ ] 基于 `aiCore` 钩子机制的标准化插件 SDK
  - [ ] 核心功能插件化（ASR、OCR、翻译等模块解耦）
  - [ ] 插件市场与动态加载机制
- [ ] **API 开放平台** - 为第三方集成提供标准接口

#### 法律场景特化

- [ ] 团队协作空间
- [ ] 案件管理系统深度集成
- [ ] 司法数据私有化部署方案

## 🏗️ 技术架构演进

### 当前架构

```
┌─────────────────────────────────────────────────────────────┐
│                      渲染进程 (Renderer)                      │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────────────┐ │
│  │   React 19   │ │  AI Core     │ │   Features           │ │
│  │   Ant Design │ │  (Vercel SDK)│ │   - Brainstorm       │ │
│  │   Tailwind   │ │              │ │   - Knowledge        │ │
│  └──────────────┘ └──────────────┘ └──────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       主进程 (Main)                          │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────────────┐ │
│  │ MCP Service  │ │  Knowledge   │ │   Agent Service      │ │
│  │ Window Mgr   │ │  Service     │ │   (Drizzle ORM)      │ │
│  │ Logger       │ │  FileSystem  │ │   Plugin System      │ │
│  └──────────────┘ └──────────────┘ └──────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 演进方向

#### 阶段一：架构与数据底座 (Foundation)

- [x] **AI Core 深度集成**：基于 Vercel AI SDK v5 重构底层模型层
- [x] **统一数据层 (LibSQL + Drizzle)**：全面迁移至主进程存储，废弃渲染进程 DB (P0)
- [ ] **插件引擎基础**：实现基于钩子 (Hooks) 的插件生命周期拦截
- [ ] **OpenAI Agents SDK**：为未来 Agentic 架构预留执行器接口

#### 阶段二：技能与业务深化 (Skills & Business)

- [x] **MCP 基础支持**：保持与官方协议兼容，提供基础工具能力
- [ ] **可视化技能编辑器**：法律从业者可直接通过 UI 编排业务逻辑
- [ ] **功能模块插件化**：将 OCR、TTS、ASR 抽离为独立插件，支持 Skill 调用
- [ ] **法律技能模版库**：内置 20+ 常用法律业务 Skill，开箱即用

#### 阶段三：智能进化 (Intelligence)

- [ ] **深度研究引擎**：实现 Deep Research 的多步推理核心控制器
- [ ] **全局 RAG 架构**：集成混合搜索 (Vector + FTS5) 与 HNSW 索引
- [ ] **GraphRAG 实现**：在 SQLite 中建立实体关系表，支持逻辑推理
- [ ] **自动化记忆管理**：实现 Agentic Memory 决策逻辑与数据压缩

#### 阶段四：跨端与协作 (不含移动端)

- [ ] 响应式 UI 适配大屏与分屏模式
- [ ] 云端同步架构 (WebDAV/S3/Cloud)
- [ ] 多人实时协作与权限控制

## 🔧 开发任务清单 (Priority List)

### 🔴 P0: 核心架构迁移与底座重构 (2026 Q3)

- [x] **统一数据层迁移**：废弃渲染进程 Dexie/Redux-Persist，全面转向主进程 LibSQL + Drizzle
- [x] **数据同步机制**：建立主进程到渲染进程的单向/双向数据同步流 (StoreSyncService 增强)
- [ ] **AI Core 完整适配**：将所有 LLM Provider 迁移至基于 Vercel AI SDK v5 的 `packages/aiCore` ⏳ **OpenAI 部分完成**
- [ ] **插件引擎基础**：实现基于钩子 (Hooks) 的插件加载与生命周期管理机制

### 🟡 P1: 智能搜索与法律业务增强 (2026 Q4)

- [ ] **混合搜索 (Hybrid Search)**：实现 LibSQL Vector + FTS5 全文搜索的 RRF 重排序算法
- [ ] **向量索引优化**：配置 LibSQL 的 HNSW 索引，提升万级知识库条目的检索速度
- [ ] **法律文书模板库**：设计并实现结构化的法律文书模板管理系统
- [ ] **法律条款检索引擎**：对接专业法规库并优化 RAG 召回精度

### 🟢 P2: 深度智能与记忆系统 (2027 Q1-Q2)

- [ ] **Agentic Memory**：开发 Agent 自主记忆提取工具，实现长效用户偏好存储
- [ ] **GraphRAG 实装**：在 SQLite 中建立实体关系图谱，支持跨文档的逻辑推理
- [ ] **深度研究引擎**：实现多步推理 (Reasoning) 与自动化搜索的任务控制器
- [ ] **插件市场/SDK**：发布标准化的插件开发规范并建立初步的插件加载中心

### ⚪ 规划中 (Planned)

- [ ] 云端同步方案 (WebDAV/S3/CRDT 冲突合并)
- [ ] 多人协作模式与权限控制系统
- [ ] 开放 API 规范定义
