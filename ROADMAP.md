# 🍒 Cherry Law 开发路线图

> 专业的法律业务场景 AI 助手开发计划

---

## 📋 概述

Cherry Law 是基于 [Cherry Studio](https://github.com/CherryHQ/cherry-studio) 二次开发的专业法律业务场景 AI 助手。本路线图规划了项目的长期发展方向和短期迭代目标。

---

## 🎯 项目愿景

成为法律从业者最信赖的 AI 助手，通过智能化工具提升法律工作效率，降低专业门槛，让法律服务更加普惠。

---

## 🗓️ 版本规划

### v1.0.0 - 语音助手版 (当前)

**状态**: 🚧 开发中  
**目标日期**: 2025 Q2

#### 核心功能

- [x] 多 LLM 服务商支持（OpenAI、Gemini、Anthropic、硅基流动等）
- [x] 本地模型部署支持（Ollama、LM Studio）
- [x] 智能助手系统（300+ 预配置助手）
- [x] 多模型同时对话
- [x] 文档处理（文本、图片、Office、PDF）
- [x] WebDAV 文件管理与数据备份
- [x] Mermaid 图表可视化
- [x] MCP（模型上下文协议）服务支持
- [x] 跨平台支持（Windows、Mac、Linux）

#### 🎙️ 语音助手功能

- [x] 语音转文字模型接入（SenseVoice 等）
- [x] 语音设置面板
- [ ] **全局快捷键触发语音输入** - 任意界面快速唤起
- [ ] **实时语音转文字** - 流式识别与反馈
- [ ] **语音指令识别** - 自然语言控制应用
- [ ] **多语言语音支持** - 中英文混合识别
- [ ] **语音质量优化** - 降噪、回声消除

#### 法律场景特化

- [ ] 法律文书模板库
- [ ] 法律条款智能检索
- [ ] 案例分析与比对
- [ ] 合同审查辅助

### v1.1.0 - 业务场景重构版 (OpenCode Agent)

**状态**: 🚧 开发中  
**目标日期**: 2025 Q3

#### 架构与引擎演进
- [x] **底层大模型引擎架构梳理**
  - [x] 确立向 Vercel AI SDK 架构 (`packages/aiCore`) 演进的整体方向
  - [x] 基于适配器模式保留对现有前端 Chunk 的兼容支持
  - [x] 清理旧版冗余提供商，仅保留 OpenAI、Anthropic、SiliconFlow 及内部内置提供商，极大地精简了底层架构
- [ ] **逐步弃用 LegacyAiProvider** - 将保留的 Provider 完全迁移至 ModernAiProvider
- [ ] **OpenCode Agent 架构重构** - 替换现有 Agent 实现逻辑
  - [ ] 基于 `@opencode-ai/sdk` 重构 Agent 核心
  - [ ] 统一 Agent 生命周期管理
  - [ ] 支持多 Agent 协作模式
  - [ ] Agent 能力动态扩展
- [ ] **业务场景引擎** - 领域特定工作流编排
  - [ ] 可视化场景构建器
  - [ ] 场景模板市场
  - [ ] 条件分支与循环支持
- [ ] **智能内容选择增强** - 精准上下文理解
- [ ] **深度研究模式** - 多源信息自动整合

#### 法律业务场景

- [ ] **法律咨询场景**
  - [ ] 智能问答工作流
  - [ ] 法规自动引用
  - [ ] 相似案例推荐
- [ ] **合同审查场景**
  - [ ] 风险点自动识别
  - [ ] 条款完整性检查
  - [ ] 修改建议生成
- [ ] **案件分析场景**
  - [ ] 证据链梳理
  - [ ] 争议焦点提取
  - [ ] 胜诉率评估
- [ ] **文书起草场景**
  - [ ] 模板智能填充

## 🏗️ 技术架构演进

### 当前架构

```
┌─────────────────────────────────────────────────────────────┐
│                      渲染进程 (Renderer)                      │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────────────┐ │
│  │   React 19   │ │  AI Core     │ │   Features           │ │
│  │   Ant Design │ │  (Vercel SDK)│ │   - Brainstorm       │ │
│  │   Tailwind   │ │              │ │   - Knowledge        │ │
│  └──────────────┘ └──────────────┘ └──────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       主进程 (Main)                          │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────────────┐ │
│  │ MCP Service  │ │  Knowledge   │ │   Agent Service      │ │
│  │ Window Mgr   │ │  Service     │ │   (Drizzle ORM)      │ │
│  │ Logger       │ │  FileSystem  │ │   Plugin System      │ │
│  └──────────────┘ └──────────────┘ └──────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 演进方向

#### 阶段一：AI Core 完善

- [x] 基于 Vercel AI SDK v5 重构
- [x] 插件系统（noThink、telemetry）
- [ ] 中间件系统完善
- [ ] OpenAI Agents SDK 预留接口

#### 阶段二：MCP 生态

- [x] MCP 服务端支持
- [x] 内置 MCP Servers（browser、filesystem、memory）
- [ ] MCP 市场/商店
- [ ] 第三方 MCP 集成

#### 阶段三：知识库增强

- [x] 文档预处理（Doc2x、Mineru、PaddleOCR）
- [ ] 向量数据库优化
- [ ] 知识图谱构建
- [ ] 多模态知识检索

#### 阶段四：多端适配

- [ ] 响应式 UI 重构
- [ ] 移动端适配层
- [ ] 云端同步架构
- [ ] PWA 支持
